# Onboarding

## Цель

Этот документ помогает новому разработчику быстро поднять проект локально и понять, где находится основная логика.

## Что нужно перед стартом

- установлен Python 3.10
- доступ к рабочей копии проекта
- возможность установить зависимости из `requirements.txt`

## Первый запуск

1. Создайте виртуальное окружение.
2. Установите зависимости.
3. Запустите тесты.
4. Найдите корневой GUI-файл проекта и убедитесь, что приложение открывается локально.

Пример команд для PowerShell:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install --upgrade pip
pip install -r requirements.txt
python -m unittest discover -s tests
```

## Куда смотреть в коде

- `core/` - инфраструктурные функции, кэш, хранение сессий
- `strategy/` - реализации торговых стратегий
- `backtest/` - движок бэктеста и метрики
- `intraday/` - интрадей-логика и smart scanner
- `optimization/` - подбор параметров
- `screening/` - сканеры тикеров и секторов
- `diary/` - дневник сделок
- `tests/` - unit-тесты по основным сценариям

## Базовый маршрут знакомства с проектом

1. Прочитать `README.md`.
2. Пройтись по `strategy/config.py` и связанным тестам.
3. Посмотреть `backtest/engine.py`.
4. Посмотреть `intraday/engine.py` и `intraday/smart_scanner.py`.
5. Запустить целевой тест перед первым изменением.

## Как безопасно вносить изменения

- менять одну логическую вещь за раз
- после изменения в торговой логике запускать релевантный модуль тестов
- перед завершением работы запускать весь набор тестов
- если меняется внешний контракт модуля, обновлять документацию

## Первые полезные команды

Запуск всех тестов:

```bash
python -m unittest discover -s tests
```

Запуск одного тестового модуля:

```bash
python -m unittest tests.test_intraday -v
```

Проверка текущих изменений git:

```bash
git status --short
```

## Что считать хорошим первым вкладом

- исправление небольшого падающего теста
- покрытие тестом незащищенной ветки логики
- уточнение README или developer docs
- небольшое упрощение модуля без изменения поведения
