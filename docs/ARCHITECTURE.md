# Architecture

Этот документ описывает высокоуровневую архитектуру проекта, зоны ответственности модулей и основные потоки данных.

## Карта модулей

- `core/` - кэширование данных, хранение сессий, базовые сервисы
- `strategy/` - стратегии и их конфигурация
- `backtest/` - движок бэктеста и расчет метрик
- `intraday/` - интрадей-движок и smart scanner
- `optimization/` - подбор параметров стратегий
- `screening/` - сканирование тикеров и секторов
- `signals/` - формирование и обработка сигналов
- `watchlist/` - избранные тикеры и связанные UI-компоненты
- `diary/` - дневник сделок и экспорт
- `analytics/` - визуализации и аналитические отчеты
- `news/`, `market/`, `monitoring/`, `automation/`, `cloud/` - прикладные вспомогательные модули

## Ключевые связи между модулями

- `strategy/` -> `backtest/` использует функции стратегии для проверки сигналов.
- `backtest/` -> `analytics/` передает метрики и результаты для визуализации.
- `intraday/` -> `strategy/` использует intraday-стратегии и метрики.
- `optimization/` -> `backtest/` выполняет серию прогонов с разными параметрами.
- `screening/` -> `backtest/` и `strategy/` для оценки сигналов по тикерам.
- `signals/` -> `screening/` и `intraday/` для нормализации выходных данных.
- `core/` используется большинством модулей как инфраструктурный слой.

## Основные потоки данных

### Бэктест

1. загрузка данных (`core/`, `market/`)
2. прогон стратегии (`strategy/` + `backtest/`)
3. метрики и графики (`backtest/` + `analytics/`)

### Интрадей-сканирование

1. загрузка данных H1 (`intraday/` + `core/`)
2. прогон стратегий (`intraday/` + `strategy/`)
3. композитная оценка (`intraday/`)
4. выдача сигналов (`signals/`)

### Оптимизация

1. подготовка сетки параметров (`optimization/`)
2. серия прогонов бэктеста (`backtest/`)
3. сортировка и выбор лучших результатов (`optimization/`)

## Контракты данных

- свечи в большинстве модулей передаются в виде списков массивов (OHLCV + даты)
- `backtest` возвращает `trades` и `metrics`
- `metrics` используется в UI и при принятии решений в smart scanner

## Примечания по расширению

- новые стратегии добавляются в `strategy/` и регистрируются в конфигурации
- новые метрики должны сопровождаться обновлением тестов и UI
- при изменениях API модулей обновляйте README и `docs/`
