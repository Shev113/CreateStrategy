=== Улучшения "Умного сканера" (SmartScanner) ===

1. ProcessPoolExecutor вместо ThreadPoolExecutor
   — настоящий параллелизм по CPU (без GIL)
   — каждый тикер обрабатывается в отдельном процессе,
     все 37 стратегий — внутри одного процесса (1 пиклинг на тикер)

2. Плавный прогресс + ETA для ProcessPoolExecutor
   — multiprocessing.Queue передаёт сигналы из воркеров
   — фоновый поток-дренаж обновляет progress_fn после КАЖДОЙ стратегии
   — средняя скорость → примерное оставшееся время (ост. 5:23 / ост. 47с)

3. Кеш ATR
   — calc_atr() вызывается 1 раз на тикер вместо 37
   — ATR передаётся во все BacktestEngine через _precomputed_atr
   — также передаётся в _calc_signal

4. _precomputed_atr в BacktestEngine
   — engine.run() использует готовый atr_series
   — исключён повторный calc_atr() внутри каждой стратегии

5. freeze_support() для PyInstaller
   — multiprocessing.freeze_support() в CreateStrategyTrading.py
   — необходимо для корректной работы ProcessPoolExecutor в exe

6. Явные импорты в strategy/__init__.py
   — все модули стратегий импортированы явно
   — исправлена ошибка "No module named 'strategy.breakout'" при сборке exe

7. Кеш стратегий в config.py (_STRATEGY_FUNC_CACHE)
   — каждый модуль стратегии импортируется 1 раз вместо 1850

8. ETA в SmartScannerUI.update_progress
   — статус показывает "245/1850 — AAPL ост. ~5:23"
   — 4-й параметр progress_fn теперь содержит строку ETA
